# Architecture Decision Record: Hallucination Auditing with DeepEval

* **Status:** Accepted
* **Last Updated:** 2026-02-19

## Context
In healthcare, a "hallucination" isn't just a bug; it is a potential medical liability that could lead to incorrect patient self-triage. We need a way to mathematically verify that the agent's summary is faithful to the original medical report.

## Decision: Implement DeepEval for Algorithmic Auditing
We have integrated DeepEval to run automated "Faithfulness" and "Answer Relevancy" tests on every agent output before it is finalized.

## Decision Rationale
* **HIPAA Liability Mitigation:** DeepEval provides a "Faithfulness Metric" (0.0 to 1.0) that ensures the summary contains zero information not present in the source document, preventing the invention of symptoms or diagnoses.
* **Automated Verification:** It allows us to unit-test the agent’s reasoning at scale, ensuring that updates to the Gemini model don't accidentally degrade medical accuracy.
* **Audit Trail for Compliance:** The scores generated by DeepEval serve as a documented "quality log," proving to auditors that the system was built with active misinformation safeguards.

## Alternative Considered
* **RAGAS** is a well-regarded alternative for evaluating RAG pipelines, but it is primarily designed for offline batch evaluation of datasets rather than the production-grade, unit-test-driven "auditor" approach required to catch hallucinations in real-time. For a HIPAA-compliant system like CliniClarity, RAGAS would fail to provide the immediate, per-request mathematical verification of faithfulness needed to mitigate medical liability and ensure that no "invented" clinical data ever reaches the patient.
